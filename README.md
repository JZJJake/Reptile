# Reptile

智能爬取 + 多级蒸馏知识库系统（FastAPI + Playwright + DeepSeek）。

## 运行

```bash
pip install -r requirements.txt
python main.py          # 监听 0.0.0.0:8000，浏览器打开 http://localhost:8000
```

## 网络访问与握手验证（可选）

当把 Reptile 暴露到局域网/公网时，可启用「握手验证」保护页面内容：

```bash
REPTILE_ACCESS_PASSWORD="你的访问密码" python main.py
```

- 登录页会多出「访问密码」一栏。客户端通过**质询/应答握手**（HMAC-SHA256，
  密码本身不在网络上明文传输）换取访问密钥（带签名、会过期的令牌，存于
  HttpOnly Cookie）。
- **连续 3 次输错访问密码即被锁定 5 分钟**，锁定期间即使输入正确密码也无法
  获取密钥，从而无法访问任何页面内容或接口。
- 未设置该环境变量时握手验证关闭，行为与本地模式一致。
- 即便在纯 HTTP 的局域网（`crypto.subtle` 不可用）下，前端也内置纯 JS 的
  SHA-256/HMAC 实现作为兜底，握手照常工作。

## 全局知识库

「知识问答」中选择 **🌐 全局知识库（跨所有库）** 即可跨所有知识库目录统一检索、
综合作答。系统会：

- 用一套**全局清单**（`wiki/_global/manifest.json`，由各库文件实时派生、可审计）
  协调各知识库内容，保证「知识库数据完好、全局数据有据可查」；
- 对每个库各自做向量检索后全局重排，取最相关页面统一作答；
- 每条信息标注其来源知识库（`[[知识库名::页面名]]`），可追溯到具体库与页面。

## 三级蒸馏管线

`wiki/wiki_manager.py`：①单篇蒸馏为知识原子 → ②组装关系网（实体归并/关系提取/
矛盾检测）→ ③关系网过大时自动主题分片（另含跨主题贯通的综合层）。
